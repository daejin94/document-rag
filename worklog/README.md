# 작업 기록 안내

작업이 중간에 중단되어도 다음 작업자가 이어받을 수 있도록 `worklog/` 디렉터리에 작업 상태를 기록한다.

## 파일 역할

- `planned.md`: 사용자가 요청했거나 작업 범위로 확정된 예정 작업
- `in-progress.md`: 현재 진행 중이거나 중단된 작업
- `done.md`: 완료된 작업

## 작업 흐름

1. 작업 시작 전 `in-progress.md`를 확인한다.
2. 진행 중이던 작업이 남아 있으면 사용자에게 이어서 진행할지 확인한다.
3. 새 작업을 시작할 때 필요한 경우 `planned.md` 또는 `in-progress.md`에 기록한다.
4. 작업이 완료되면 `in-progress.md`의 항목을 `done.md`로 이동한다.
5. 완료되지 않은 작업은 `in-progress.md`에 남겨 다음 작업자가 이어받을 수 있게 한다.

## planned.md 기록 기준

`planned.md`에는 사용자가 요청했거나 작업 범위로 확정된 항목만 기록한다.

기록하는 항목:

- 사용자 요청으로 진행하기로 한 작업
- 현재 작업 범위에 포함된 후속 처리
- 진행 전 정리가 필요한 확정 작업

기록하지 않는 항목:

- 추후 개발 아이디어
- 있으면 좋을 기능
- 사용자 요청 없는 개선 후보
- 우선순위가 정해지지 않은 백로그
- AI coding agent가 임의로 제안한 기능

## 작성 원칙

- `planned.md`, `in-progress.md`, `done.md`에는 실제 작업 항목만 기록한다.
- 작업 기록은 필요한 내용만 간단히 작성한다.
- 실제 코드 변경 범위와 무관한 내용을 추가하지 않는다.
- 사용자의 요청 범위를 벗어난 기능을 임의로 작업 항목에 추가하지 않는다.

## 항목 형식

```md
## 작업명

- 시작일:
- 목적:
- 현재 상태:
- 다음 작업:
- 관련 파일:
```
